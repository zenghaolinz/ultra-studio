# Ultra Studio 用户手册

## 1. 应用简介

Ultra Studio 是一个本地 AI 创作工作台，支持 Agent 对话、图片生成、图片改进、3D 资产生成、模型预览、生成历史和运行诊断。它适合用于课程作品、文创资产、游戏原型、工业概念件和个人创作流程。

## 2. 启动应用

在项目根目录运行：

```powershell
.\start.ps1
```

首次启动时脚本会检查 Node、Python 依赖，并在缺少 `sidecar/config.ini` 时从 `sidecar/config.example.ini` 自动复制一份。

如果需要手动启动：

```powershell
npm install
python -m venv sidecar\.venv
sidecar\.venv\Scripts\pip.exe install -r sidecar\requirements.txt
npm run tauri dev
```

## 3. 配置 ComfyUI

打开 `sidecar/config.ini`，填写本机 ComfyUI Windows Portable 路径：

```ini
[ComfyUI]
host = 127.0.0.1
port = 8188
path = E:/ComfyUI_windows_portable
```

如果不配置 ComfyUI，聊天和部分文件能力仍可使用，但图片生成和 3D 生成不可用。

## 4. 运行诊断

启动应用后进入 `设置 -> 诊断`。

诊断页会检查：

- Python Sidecar 是否响应
- SQLite 本地数据库是否可访问
- 聊天模型和 Embedding 模型是否配置
- `sidecar/config.ini` 是否存在
- ComfyUI 路径和运行状态
- ComfyUI 输出目录是否可访问

如果出现警告或错误，优先按诊断项中的提示处理。

## 5. Agent 对话

在 Agent 工作区中输入自然语言需求。常见用法包括：

- 询问项目使用方法
- 让 Agent 生成图片或 3D 资产
- 让 Agent 整理文件、创建文档、读取 PDF
- 让 Agent 根据已有素材提出修改建议

对于删除、覆盖等危险操作，应用会根据当前模式进行确认或限制。

### 连续任务编排

Agent 支持一次描述包含依赖关系的创作任务，例如：

```text
生成一张飞船的图片，并生成三视图，然后生成模型
```

系统会先生成源图，在获得实际图片文件后继续生成三视图，最后用已知视角图片重建 3D 模型。

## 6. 图片生成与改进

进入图片工作区后，可以输入提示词生成概念图。建议提示词包含：

- 主体对象
- 材质风格
- 使用场景
- 颜色和细节
- 是否适合转为 3D

示例：

```text
一个校园文创机器人摆件，金属与磨砂塑料材质，蓝白配色，圆润结构，适合转为 3D 模型
```

生成后可以继续使用图片改进功能调整风格、颜色和细节。

## 7. 3D 资产生成

进入 3D 工作台后，可以使用三种方式：

- 文字生成 3D：输入描述后直接生成模型。
- 图片转 3D：上传或选择一张概念图生成模型。
- 双图融合 3D：上传两张参考图并输入融合描述。

质量模式：

- `快速`：适合现场演示和快速预览。
- `高质量`：适合最终输出，但耗时更长。

生成完成后，可以在预览区查看模型、2D 渲染图、法线图和 UV 贴图。

### 修改已有模型外观

当前系统没有直接编辑 GLB 网格或材质参数的能力。输入：

```text
把刚才这个模型改成金属材质，生成三视图，然后重建模型
```

系统实际会修改上一模型关联的源图或预览图，基于修改后的图片生成三视图，再输出一个重新生成的新模型。结果中显示的 `重建源图` 用于说明新模型所依据的视觉输入。

## 8. 生成历史

3D 工作台左侧的 `生成历史` 会记录最近任务，包括：

- 概念图生成
- 图片改进
- 文字生成 3D
- 图片转 3D
- 双图融合 3D
- 展示材料生成

可用操作：

- `复用`：把历史提示词和质量模式恢复到当前输入。
- `恢复`：把历史输出恢复到当前预览区。
- `打开位置`：在文件管理器中打开输出文件位置。

如果现场生成耗时较长，可以先用生成历史展示已完成结果。

## 9. 展示材料

完成一个模型后，可以生成展示材料，用于记录：

- 作品定位
- 生成需求
- 产物清单
- 技术亮点
- 演示步骤

默认材料会保存到 `docs/showcase/`。

## 10. 常见问题

### ComfyUI 未就绪

检查 `sidecar/config.ini` 中的路径、端口和 ComfyUI 是否已经启动。然后进入 `设置 -> 诊断` 查看具体错误。

### 生成很慢

首次加载模型会比较慢。现场演示建议使用 `快速` 模式，并提前准备生成历史样例。

### PowerShell 显示中文乱码

项目源码按 UTF-8 保存。读取中文文件时使用：

```powershell
Get-Content -Encoding UTF8 文件路径
```

### 构建时出现 chunk 过大警告

只要 `npm run build` 最终成功，该警告不影响当前桌面应用提交和演示。
